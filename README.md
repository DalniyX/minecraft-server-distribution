# 🚀 Minecraft Modpack Utility Launcher — Документация Проекта

Премиальный гибридный лаунчер для Minecraft, сочетающий в себе производительность и низкоуровневые возможности **Python** с гибким, красивым и анимированным интерфейсом на **React (TypeScript)**. Проект разработан для автоматической синхронизации модификаций «по воздуху» (OTA) и удобного администрирования сборок через GitHub API и интеграцию с ИИ (Gemini).

## 🏗 ТЕХНОЛОГИЧЕСКИЙ СТЕК И АРХИТЕКТУРА

Лаунчер функционирует по принципу гибридного десктопного моста:

```
┌────────────────────────────────────────────────────────────────────────┐
│                      СЛОЙ ПРЕДСТАВЛЕНИЯ (React)                        │
│             Интерфейс, Админ-панель, Логика тем, Настройки             │
└──────────────────────────────────┬─────────────────────────────────────┘
                                   │
                     Асинхронные вызовы через мост
                   (window.pywebview.api / JS Inject)
                                   │
┌──────────────────────────────────▼─────────────────────────────────────┐
│                       СИСТЕМНОЕ ЯДРО (Python)                          │
│            Файловые операции, Потоки загрузки, Локальный кэш           │
└────────────────────────────────────────────────────────────────────────┘
```

1. **Frontend (React 18 + Vite + Tailwind CSS):** Отвечает за рендеринг, анимации переходов, локализацию, таблицы модов, ИИ-интеграцию и работу с GitHub REST API.
2. **Backend (Python 3.11+ + Pywebview):** Выступает в качестве нативного контейнера. Вычисляет контрольные хэши SHA-256 локальных файлов, выполняет многопоточную загрузку, открывает папки в проводнике Windows и управляет положением кастомного безрамочного окна (`drag_window`).

## 📂 ПОДРОБНАЯ КАРТА И МОДУЛЬНАЯ СТРУКТУРА ФАЙЛОВ

Проект разделен на две независимые части, объединенные в единый исполняемый процесс.

### 🧩 1. Python-Бэкенд (Системное ядро)
* **`main.py`** — Точка входа в приложение. Создает нативное десктопное окно, пробрасывает класс взаимодействия `Api` во фронтенд, отвечает за кастомный драг окна и интеграцию с API Modrinth.
* **`core/`** — Модули для работы с локальной системой:
    * **`config_manager.py`** — Управляет локальным конфигом `config.json` в `%LOCALAPPDATA%/DalniyX_Launcher`. Хранит локальные пути, токен GitHub и ключи ИИ.
    * **`file_manager.py`** — Системный сканер. Читает директорию `.minecraft`, вычисляет хэши модов, очищает устаревшие файлы.
* **`workers/`** — Фоновые потоки выполнения задач:
    * **`download_worker.py`** — Многопоточный воркер загрузки (до 5 параллельных потоков) с поддержкой докачки файлов через HTTP-заголовки `Range`. Транслирует скорость, процент выполнения и остаток времени в React через JS-инъекции.

### ⚛️ 2. React-Фронтенд (Интерфейс)
* **`src/main.tsx`** — Точка инициализации React-приложения в DOM.
* **`src/App.tsx`** — Корневой менеджер экранов. Переключает интерфейс между стартовым заставкой (`SplashScreen`), панелью игрока (`PlayerPanel`) и админкой (`AdminPanel`).
* **`src/types.ts`** — Единый реестр TypeScript-типов. Описывает структуры файлов, манифеста, конфигурации GitHub и загрузчиков.
* **`src/pywebview_mock.ts`** — Эмулятор бэкенда для веб-браузера. Позволяет верстать и тестировать интерфейс в Chrome/Firefox без запуска Python-процесса.
* **`src/index.css`** — Главный CSS-файл. Содержит конфигурацию Tailwind и правила стилизации матового стекла (Frosted Glass).

#### 🗂 Папка `src/components/` (Визуальные компоненты)
* **`TitleBar.tsx`** — Безрамочная верхняя панель управления окном (свернуть, закрыть, перетащить).
* **`SplashScreen.tsx`** — Экран загрузки. Проверяет связь и настраивает базовые пути.
* **`PlayerPanel.tsx`** — Основной экран игрока. Содержит кнопку «ИГРАТЬ», вкладку с модами и чейнжлог обновлений.
* **`SidebarStats.tsx`** — Статистическая боковая панель (количество модов, общий размер, версия MC).
* **`ModpackTable.tsx`** — Таблица файлов сборки с возможностью выборочного отключения модов и открытия их папок на диске.
* **`DownloadProgress.tsx`** — Визуальный трекер многопоточного скачивания с индивидуальными шкалами по каждому потоку.
* **`LoginModal.tsx`** — Форма авторизации администратора.
* **`UserSettingsModal.tsx`** — Настройки игрока: переключение языка, темы (Светлая/Темная) и путей к директории игры.

#### 🗂 Папка `src/components/admin/` (Панель администратора)
* **`AdminHeader.tsx`** — Верхняя панель администрирования со статусом синхронизации.
* **`AdminFilesTab.tsx`** — Управление файлами. Поддерживает Drag-and-Drop загрузку локальных `.jar` файлов, а также встроенный поиск и скачивание модов напрямую из Modrinth API.
* **`AdminSettingsTab.tsx`** — Настройка интеграции с GitHub, выбор версий Minecraft, кастомизация оформления сборки (замена фонового изображения, изменение цвета акцентов) и генерация чейнжлогов с помощью встроенного ИИ Gemini.

## 🎨 ДИЗАЙН-СИСТЕМА И ОФОРМЛЕНИЕ (GLASS DESIGN SYSTEM)

Лаунчер разработан с учетом правил высокой контрастности и стильного матового оформления для обеих тем.

### 🌗 1. Глобальные CSS переменные
Переменные управляются динамически через добавление/удаление класса `.light` на корневой тег `<html>`:

```css
:root {
  /* Темная тема (Deep Cosmic Minimal) */
  --bg-app: #08080c;
  --bg-panel: rgba(22, 22, 32, 0.58); /* Плотное серое стекло */
  --bg-panel-secondary: rgba(30, 30, 44, 0.72);
  --text-main: #e2e2e9;
  --text-muted: rgba(255, 255, 255, 0.45);
  --text-title: #ffffff;
  --border-main: rgba(255, 255, 255, 0.08);
}

.light, .light-mode {
  /* Светлая тема (Slate & White Glass) */
  --bg-app: #cbd5e1; /* Приглушенный благородный серый */
  --bg-panel: rgba(255, 255, 255, 0.82); /* Высокоплотное светлое матовое стекло */
  --bg-panel-secondary: rgba(244, 248, 252, 0.88);
  --text-main: #1e293b; /* 100% контрастный темно-серый */
  --text-muted: #475569; /* Серый сланец */
  --text-title: #0f172a; /* Глубокий темно-синий */
  --border-main: rgba(15, 23, 42, 0.12);
}
```

### ✨ 2. Особенности реализации матового стекла (Frosted Glass)
* **Для Темной Темы:** Легкое размытие заднего плана (`backdrop-filter: blur(24px)`) на полупрозрачной серой подложке. Позволяет фоновому изображению мягко просвечивать сквозь панели.
* **Для Светлой Темы:** Усиленное матовое размытие повышенной плотности (`backdrop-filter: blur(28px)`) с белой границей (`rgba(255,255,255,0.65)`). Изолирует текст от пестрого фона, обеспечивая безупречную читаемость в соответствии со стандартами WCAG AA.

## 🛠 РУКОВОДСТВО ПО НАСТРОЙКЕ И РАЗРАБОТКЕ В INTELUIJ IDEA

Если вы импортировали проект в IntelliJ IDEA / WebStorm, выполните следующие шаги для подготовки рабочего окружения.

### 1. Подготовка окружения Node.js (Frontend)
1. Откройте встроенный терминал в IntelliJ IDEA (`Alt + F12`).
2. Установите зависимости проекта:
   ```bash
   npm install
   ```
3. Запустите локальный сервер разработки фронтенда:
   ```bash
   npm run dev
   ```

### 2. Подготовка окружения Python (Backend)
1. Убедитесь, что в IntelliJ IDEA установлен плагин **Python** (или используйте PyCharm).
2. Настройте локальное виртуальное окружение (Virtual Environment) в настройках проекта:
   `File | Settings | Project: minecraft_modpack_tool | Python Interpreter`. Нажмите *Add Interpreter* и выберите *Virtualenv Environment*.
3. Установите зависимости бэкенда через терминал с активированным venv:
   ```bash
   pip install -r requirements.txt
   ```

### 3. Запуск приложения в десктопном режиме
Для запуска полноценного десктопного приложения (с нативными окнами):
```bash
python main.py
```

## 🧭 ПОШАГОВЫЕ ИНСТРУКЦИИ ДЛЯ ПОЛЬЗОВАТЕЛЯ

### 🎮 А. Инструкция для Игрока (Пользовательский сценарий)
1. **Запуск:** Откройте лаунчер. Нажмите кнопку «Настройки» (шестеренка в левом нижнем углу) для выбора папки Minecraft и смены языка/темы при необходимости.
2. **Синхронизация:** На главном экране нажмите кнопку **«СИНХРОНИЗИРОВАТЬ И ИГРАТЬ»**.
3. **Процесс:** Лаунчер автоматически сверит хэши ваших локальных модов с сервером. Если обнаружатся устаревшие или лишние файлы, они будут автоматически заменены или удалены.
4. **Старт:** После завершения скачивания кнопка сменится на «ЗАПУСТИТЬ ИГРУ». Приятной игры!

### 👑 Б. Инструкция для Администратора (Создателя сборки)
1. **Вход:** Нажмите кнопку **«Войти в админ-панель»** в правом верхнем углу лаунчера, введите секретный пароль (задается в `manifest.json`).
2. **Конфигурация репозитория:** Перейдите во вкладку **«Конфигурация»** и укажите данные вашего GitHub-репозитория и токен доступа (PAT) для публикации файлов. Нажмите «Сохранить».
3. **Добавление модов:**
    * **Локально:** Перейдите во вкладку «Манифест и Моды» и просто перетащите файлы `.jar` в область загрузки.
    * **Через Modrinth:** Введите имя мода в поиске. Система отфильтрует версии под вашу сборку. Нажмите «Скачать» — лаунчер самостоятельно скачает мод, рассчитает его размер и хэш SHA-256.
4. **Публикация сборки:** Нажмите кнопку **«Опубликовать Моды»**. Лаунчер автоматически загрузит новые файлы в ваш репозиторий и перезапишет `manifest.json`. Все ваши игроки получат обновление модов при следующем запуске приложения!
